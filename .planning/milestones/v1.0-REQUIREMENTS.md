# Requirements: PLBeast

**Defined:** 2026-06-18
**Core Value:** Accurately predict and display the next beast in the Pack Leader rotation

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Core Tracking

- [ ] **TRACK-01**: Addon correctly tracks the cyclic beast rotation order (wyvern → boar → bear → wyvern)
- [ ] **TRACK-02**: Rotation advances when a beast actually spawns, detected via `C_UnitAuras.GetPlayerAuraBySpellID` on ready buff spell IDs
- [ ] **TRACK-03**: The saved next-beast prediction persists via SavedVariablesPerCharacter and is preserved across a `/reload`; it is reset to wyvern on a fresh login / full relog and on pulling a raid/dungeon boss (ENCOUNTER_START), while zoning and trash leave the prediction unchanged _(NOTE: the login/boss-pull reset clause is **superseded-pending** by Phase 5.1, which adopts AzortharionUI's self-correcting model and drops these explicit resets for parity; re-adding them is deferred to a post-5.1 decision — see PERF-01 and `.planning/todos/pending/2026-06-24-reconsider-login-boss-reset.md`)_
- [ ] **TRACK-04**: Addon supports both Beast Mastery and Survival hunter specs
- [ ] **TRACK-05**: Rotation state uses snapshot-diff pattern (only advances on buff transition, not every poll tick)

### Visibility

- [ ] **VIS-01**: Icon only displays when Pack Leader hero talent is active (check via `IsPlayerSpell` with sentinel/dark ranger exclusion)
- [ ] **VIS-02**: Icon hides when player spec does not support Pack Leader (MM, non-hunter)
- [ ] **VIS-03**: Icon hides/shows correctly on spec change (`PLAYER_SPECIALIZATION_CHANGED`) and talent change events

### UI

- [x] **UI-01**: Single icon displays the next beast's texture (wyvern/boar/bear)
- [x] **UI-02**: Icon has a square border using BackdropTemplate (WHITE8X8 edge texture) with configurable color and thickness; default: black, 1px
- [x] **UI-03**: Icon dragging is toggled on/off (not always draggable); blocked during combat lockdown
- [x] **UI-04**: Icon position persists across sessions via SavedVariables
- [x] **UI-05**: Icon width and height are independently adjustable _(sync toggle was built in Phase 4, then removed in Phase 5 by user decision — dimensions are always independent)_
- [x] **UI-07**: Icon width and height persist across sessions _(sync-toggle setting removed in Phase 5)_
- [x] **UI-06**: Border color and thickness persist across sessions

### Configuration

- [x] **CFG-01**: Slash command `/plbeast` registered as entry point, opens options frame
- [x] **CFG-02**: Lightweight options frame with sliders for width, height, border thickness _(border color picker removed by user decision during Phase 5 — border is a fixed black outline)_
- [x] **CFG-03**: Options frame includes drag (lock) toggle _(width/height sync toggle removed by user decision during Phase 5 — not built)_
- [x] **CFG-04**: Options frame blocked from opening during combat lockdown

### Addon Structure

- [x] **STRUCT-01**: Root repo contains a `PLBeast/` folder with the complete addon inside (standard WoW addon packaging)
- [x] **STRUCT-02**: Code extracted/adapted from PackLeaderHelper — not written from scratch
- [x] **STRUCT-03**: Proper `.toc` file targeting Midnight (12.x) with correct Interface version
- [x] **STRUCT-04**: SavedVariablesPerCharacter declaration in `.toc`
- [x] **STRUCT-05**: Localization support (at minimum enUS)

### Performance

- [x] **PERF-01**: Beast detection and rotation advancement use per-frame CDM poll (OnUpdate with `GetTime()` throttle, matching AzortharionUI's `Hunter.Update()` pattern) with no recurring `C_Timer` poll ticker — negligible idle CPU/GC cost — while preserving rotation behavior and the no-freeze guarantee

### Release

- [x] **REL-01**: GitHub Actions workflow that packages the `PLBeast/` folder into a release zip
- [x] **REL-02**: Release triggered by git tag push (e.g. `v1.0.0`)
- [x] **REL-03**: Release zip structured correctly for WoW addon installation (unzips to `PLBeast/`)
- [x] **REL-04**: Repo linked to `cela28/PLBeast` as remote origin

Deferred to future release. Tracked but not in current roadmap.

### Localization

- **LOC-01**: Chinese localization (zhCN)

### Text Display Mode

- [ ] **TEXT-01**: A text-based display mode shows each beast as a colored text label instead of an icon texture
- [ ] **TEXT-02**: Each beast has a distinct, readable color (wyvern, boar, bear) that is clearly distinguishable
- [ ] **TEXT-03**: The text display updates in real-time as the rotation prediction changes, using the same state machine as icon mode
- [ ] **TEXT-04**: User can toggle between icon mode and text mode via the options frame or slash command
- [ ] **TEXT-05**: Text mode settings (font size, position) persist across sessions via SavedVariables
- [ ] **TEXT-06**: Text display respects the same visibility gating as icon mode (Pack Leader talent, BM/SV spec)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Cooldown timer / howl-CD tracking | Stays in PackLeaderHelper |
| Wyvern buff duration tracking | Stays in PackLeaderHelper |
| Hogstrider tracking | Stays in PackLeaderHelper |
| Grid layout editor | PLBeast is a single icon, not a multi-icon tracker |
| ~~Multiple display modes (bars, text)~~ | ~~WeakAuras handles this better; stay opinionated~~ — **Rescoped**: text mode added as Phase 7 (user request 2026-07-07) |
| Sound alerts | Unnecessary for a glanceable indicator |
| Rotation suggestion / "cast this" | Banned by Blizzard in Midnight; display state only |
| Minimap button | Slash command is sufficient |
| Party/raid sync | Pack Leader is solo-rotation mechanic |
| Settings import/export | Overkill for position + scale + border settings |
| Modifying PackLeaderHelper | This project is PLBeast only |
