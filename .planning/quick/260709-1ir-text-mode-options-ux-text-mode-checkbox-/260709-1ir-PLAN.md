  <action>
Restructure the options frame so control visibility follows DB.textMode, with the Text Mode checkbox on top and a new Outline cycle-control. DO NOT modify ApplyDisplayMode (~L555) — its ApplyIconSettings delegation must stay intact.

APPROACH — least-risky WoW 12.0.x choice (no Ace3/libs): implement Outline as a small custom cycle-BUTTON (UIPanelButtonTemplate), not a UIDropDownMenu/menu-API dropdown. The button label reads "<Outline>: <current style>" and each click advances None -> Outline -> Thick Outline -> None. This is trivial to SetShown/SetPoint, has no template/menu-API churn risk, and is reachable only when Text Mode is ON (relayout hides it otherwise). The task brief explicitly permits a cycle-button.

1. Store control handles. In ToggleOptions, capture the return value of every Create* call into `optionsFrame.controls` fields so a relayout routine can reach them:
   - `controls.textMode` (the Text Mode checkbox)
   - `controls.lock` (Lock position checkbox)
   - `controls.width`, `controls.height`, `controls.border` (the three icon-mode sliders)
   - `controls.fontSize` (Font Size slider)
   - `controls.outline` (new Outline cycle-button, see step 4)
   - `controls.wyvern`, `controls.boar`, `controls.bear` (the three color swatches)
   The creation-time yOffset arguments become placeholders — RelayoutOptions is authoritative for final positions — so leave existing yOffset args as-is (they are overwritten) or pass 0; do not delete the calls.

2. Fix parent-owned label hiding (IMPORTANT gotcha). `CreateCheckbox` stores its label as `check.text` (parented to the OPTIONS FRAME, not the checkbox), `CreateSlider` stores `slider.valText` (parent-owned), and `CreateColorSwatch`'s label is parent-owned but NOT stored. Because these fontstrings are children of optionsFrame, calling `SetShown(false)` on the widget will NOT hide them — they float as orphans. To fix:
   - In `CreateColorSwatch` (~L731-734), store the label on the button: `button.label = text` (add this line; the helper already returns button).
   - Do not change how checkbox `.text` / slider `.valText` are created; the relayout helper (step 3) will hide them explicitly.

3. Add a file-local `RelayoutOptions(frame)` function defined BEFORE ToggleOptions (so ToggleOptions and the Text Mode callback can call it). Include a small local helper `SetControlShown(w, shown)` that does `w:SetShown(shown)` and also `w.valText`, `w.text`, `w.label` if present (covers slider value string, checkbox label, swatch/outline label). RelayoutOptions logic:
   - Read `local c = frame.controls` and `local textMode = DB.textMode`.
   - Maintain a running `y` cursor starting at about -40 (below the title bar). Anchor each visible control TOPLEFT to `frame` with ClearAllPoints + SetPoint, then advance `y`.
   - ALWAYS place first, in this order: `c.textMode` (x=16, checkbox), then `c.lock` (x=16, checkbox). Advance ~30px per checkbox.
   - If textMode: show and stack `c.fontSize` (x=20 slider), `c.outline` (x=20 button), then `c.wyvern`/`c.boar`/`c.bear` (x=20 swatches); hide `c.width`/`c.height`/`c.border`. Sliders need ~14px top headroom for their `.Text` label plus ~34px below (≈48px total); button ~30px; swatches ~24px each.
   - Else (icon mode): show and stack `c.width`/`c.height`/`c.border` (x=20 sliders, ~48px each); hide `c.fontSize`/`c.outline`/`c.wyvern`/`c.boar`/`c.bear`.
   - Goal: no vertical gaps from hidden controls and no overlap; tune the pixel advances so labels/value-strings don't collide (frame is 280x460 — ample room).

4. Add the Outline cycle-button. Create it inline in ToggleOptions (or a small `CreateOutlineControl(parent)` helper) as a `CreateFrame("Button", nil, parent, "UIPanelButtonTemplate")`, width ~150, height ~22. Give it its OWN fontstring OR use the template's built-in text via `:SetText(...)` (template text is a child of the button, so it hides with the button — preferred; if you add a separate fontstring, parent it to the button and store as `.label`). Define an ordered list of the three styles as `{value, labelKey}` pairs: empty-string/None, standard-outline-flag/Outline, thick-outline-flag/Thick Outline. Helper `SetOutlineText()` sets the button text to `L["Outline"] .. ": " .. L[currentStyleLabel]` based on DB.textOutline. On click: advance to the next style (wrapping), set `DB.textOutline`, apply live to the font — `local file, size = textFont:GetFont(); textFont:SetFont(file, size, DB.textOutline)` — then, if `DB.textMode and root and root.label`, re-measure the frame exactly like the Font Size slider does (~L889-893: SetSize from GetStringWidth+4 / GetStringHeight+4, clamped to min 20/16), then call SetOutlineText() to refresh the label. Initialize the button text via SetOutlineText() at creation. Store the button as `controls.outline`.

5. Move Text Mode to top + wire relayout triggers:
   - The Text Mode checkbox already exists (~L866-875) with a setValue that does `DB.textMode = checked; ApplyDisplayMode()`. Append `RelayoutOptions(optionsFrame)` after ApplyDisplayMode() so toggling re-lays-out immediately. Its on-top position is handled by RelayoutOptions (step 3), so physical creation order need not change — but you MAY reorder its Create call earlier for readability.
   - At the end of ToggleOptions, in the branch that SHOWS the frame (the `else` that calls `optionsFrame:Show()`), call `RelayoutOptions(optionsFrame)` BEFORE `optionsFrame:Show()` so visibility is correct every open. Also call it once right after the initial lazy build (before the initial `optionsFrame:Hide()`), so first open is correct.

Preserve: no slash-command toggle for text mode (do not add one). Keep 1-tab indentation, PascalCase function names (RelayoutOptions, SetControlShown / CreateOutlineControl), and guard returns (`if not frame or not frame.controls then return end` at top of RelayoutOptions).
