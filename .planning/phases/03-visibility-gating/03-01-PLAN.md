<objective>
Add Pack Leader hero talent detection and spec-based visibility gating to PLBeast. After this plan, the addon conditionally registers UNIT_AURA only when the player is on a BM or SV hunter with Pack Leader active. Talent and spec changes mid-session trigger a deferred visibility refresh that correctly registers or unregisters aura tracking.

Purpose: Gate rotation tracking to relevant specs/talents so the addon does not process irrelevant aura events on alts, MM hunters, or when a different hero tree is selected (VIS-01, VIS-02, VIS-03). This flag becomes the single source of truth that Phase 4's icon UI reads for show/hide.

Output: Modified PLBeast.lua with IsPackLeaderHeroTalent(), RefreshVisibility(), QueueVisibilityRefresh(), conditional UNIT_AURA registration, and five talent/spec event handlers.
</objective>

## Phase Goal

**As a** Pack Leader hunter player, **I want to** have PLBeast automatically detect when Pack Leader is my active hero talent, **so that** rotation tracking only runs when relevant and hides when I switch to a non-Pack Leader spec or talent.

| Boundary | Description |
|----------|-------------|
| WoW Lua Sandbox | All addon code runs inside Blizzard's sandboxed Lua VM; no filesystem, network, or system access |
| IsPlayerSpell API | Returns boolean from WoW client's internal spell database; not controllable by other addons |
| Event system | WoW fires events; addon cannot forge events but must handle unexpected event arguments gracefully |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | SPELL_HOTPL_PARENT constant | accept | Constant is a local; no other addon can modify it. If Blizzard changes the spell ID, the addon fails safe (isPackLeaderActive = false) |
| T-03-02 | Denial of Service | QueueVisibilityRefresh burst | mitigate | pendingVisibilityRefresh flag coalesces rapid talent events into a single deferred call; prevents unbounded C_Timer.After queue growth |
| T-03-03 | Information Disclosure | dprint debug output | accept | Debug output is gated by DB.debug toggle; visible only to the local player in their chat window. No sensitive data exposed — only spec name and boolean flag |
| T-03-04 | Tampering | Global namespace | mitigate | No new globals added; isPackLeaderActive, pendingVisibilityRefresh, and all functions are file-local. Only existing globals (PLBeastDB, PLBeastLocale) unchanged |
| T-03-05 | Elevation of Privilege | IsPlayerSpell stale read | mitigate | C_Timer.After(0, ...) deferral in QueueVisibilityRefresh ensures IsPlayerSpell reads updated talent state, not stale pre-event data |
