## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Slash command input | User-typed /plbeast subcommands — untrusted string input |
| WoW API return values | C_UnitAuras.GetPlayerAuraBySpellID may return nil or unexpected structures |
| SavedVariables load | PLBeastDB loaded from disk may contain corrupted or out-of-range values |
| UNIT_AURA event args | unitTarget argument could be any unit string, not just "player" |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | SavedVariables (DB.nextIndex) | mitigate | NormalizeNextIndex() clamps to 1-3 range with tonumber fallback on every load; corrupted values resolve to index 1 (wyvern) |
| T-02-02 | Denial of Service | UNIT_AURA event handler | mitigate | Guard unitTarget ~= "player" immediately; return before any processing for non-player units. Prevents unnecessary computation in group content |
| T-02-03 | Denial of Service | C_UnitAuras API unavailability | mitigate | IsReadyBuffActive guards with if not (C_UnitAuras and C_UnitAuras.GetPlayerAuraBySpellID) then return false end; graceful degradation to "no buff active" |
| T-02-04 | Tampering | Slash command input | mitigate | Input normalized via :lower():match("^%s*(.-)%s*$"); only exact string matches ("debug", "reset") route to handlers; all other input falls through to usage help |
| T-02-05 | Information Disclosure | Debug output in chat | accept | Debug output only shows addon-internal state (beast names, boolean flags, index). No PII, no secrets. Gated behind explicit DB.debug toggle |
| T-02-06 | Tampering | Aura data (expirationTime, duration) | mitigate | Nil guards on all aura table fields before arithmetic: (aura and aura.expirationTime and aura.duration) and (...) or 0. Prevents Lua errors on unexpected WoW API returns |
</threat_model>
