# Phase 5 ‚Äî UI Design Contract: Configuration

> Visual and interaction contract for the PLBeast options frame.
> This is a World of Warcraft addon ‚Äî all UI is built with the Blizzard Frame API (Lua/WoW
> templates). There is no HTML, CSS, or web component library. "Design system" means Blizzard
> native templates and font objects. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

WoW frame layout uses pixel values, not abstract tokens. All control placement uses
multiples of 8 pixels on the vertical axis.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4 px | Gap between label text and swatch; swatch internal padding |
| sm | 8 px | Spacing between checkbox and its label; between editBox and slider |
| md | 16 px | Left inset from frame edge (`TOPLEFT` x-offset); gap between slider bottom and next row |
| lg | 24 px | Blank row separating sliders from toggles |
| title | 40 px | First control `yOffset` below the frame top (clears title bar + inset area) |

**Vertical control stride:** Each slider row occupies 48 px of vertical space (16 px
slider height + 8 px valText gap + 8 px bottom breathing room + 16 px to next control
top). Each checkbox row occupies 32 px.

**Control y-offsets (TOPLEFT from frame origin, negative = downward):**

| Control | y-offset |
|---------|----------|
| Width slider | -48 |
| Height slider | -96 |
| Border thickness slider | -144 |
| Border color label + swatch | -200 |
| Lock position checkbox | -228 |
| Sync width & height checkbox | -252 |

Exceptions: color swatch is 20 √ó 20 px ‚Äî sized for mouse click target, not on the 8 pt grid.

| Role | Value | Usage |
|------|-------|-------|

**Accent reserved for:** The border color swatch only ‚Äî it is the single element that

## Control Specifications

### Sliders (3 total)

All sliders use `OptionsSliderTemplate`. Width 230 px, height 16 px.
Attached `valText` (`GameFontHighlightSmall`) positioned `TOP, slider, BOTTOM, 0, -2`.
No editBox ‚Äî PLBeast trims PLH's editBox from `CreateSlider` to keep the widget minimal
(editBox adds 70 px horizontal width; the PLBeast frame is narrower).

| Control | Label | Min | Max | Step | DB Field | Setter |
|---------|-------|-----|-----|------|----------|--------|
| Width slider | `L["Width"]` | 16 | 128 | 1 | `DB.width` | `SetIconSize(value, DB.height)` |
| Height slider | `L["Height"]` | 16 | 128 | 1 | `DB.height` | `SetIconSize(DB.width, value)` |
| Border thickness slider | `L["Border Thickness"]` | 0 | 8 | 1 | `DB.borderThickness` | `ApplyIconSettings()` after writing `DB.borderThickness = value` |

All sliders are integer-valued (`isInteger = true` in the helper).

**Height slider dimming:** When `DB.syncSize = true` the height slider is visually
dimmed (alpha 0.4, `EnableMouse(false)`) because it mirrors the width automatically.
The height slider re-enables when `DB.syncSize = false`. Dimming is applied immediately
when the sync toggle is clicked. Source: PLH `SetSliderEnabled` (line 652).

### Color Picker (1 swatch + native dialog)

| Property | Value |
|----------|-------|
| Swatch frame | `CreateFrame("Button", nil, optionsFrame)`, size 20 √ó 20 |
| Swatch texture | `"Interface\\Buttons\\WHITE8X8"` colored via `SetVertexColor(r, g, b, a)` |
| Swatch anchor | `TOPLEFT, optionsFrame, TOPLEFT, 16, -200` (right of "Border Color:" label) |
| Click behavior | Calls `OpenColorPicker({ ... })` (modern table-arg API, 12.x) |
| Section label | `GameFontHighlight` FontString: `"Border Color:"`, anchored left of swatch |
| `hasOpacity` | `true` ‚Äî exposes the alpha slider (CONTEXT.md D-02) |
| `swatchFunc` | Write `DB.borderColor = {r,g,b,a}` then `ApplyIconSettings()` |
| `opacityFunc` | Write `DB.borderColor.a = opacity` then `ApplyIconSettings()` |
| `cancelFunc` | Restore the saved-before-open `{r,g,b,a}` snapshot then `ApplyIconSettings()` |

**API note:** Use `OpenColorPicker({r, g, b, opacity, hasOpacity, swatchFunc, opacityFunc, cancelFunc})`
(the modern 12.x table-arg form). Seed `r/g/b` from `DB.borderColor.r/g/b`; seed
`opacity` from `1 - DB.borderColor.a` (WoW passes opacity as `1 - alpha`). Source:
CONTEXT.md D-01, D-02. This widget is net-new; no in-repo precedent.

### Checkboxes (2 total)

All checkboxes use `UICheckButtonTemplate`. Anchored `TOPLEFT, optionsFrame, TOPLEFT,
16, yOffset`.

| Control | Label | DB Field | Setter | Default |
|---------|-------|----------|--------|---------|
| Lock position | `L["Lock position"]` | `DB.locked` | Write `DB.locked = checked` (no ApplyIconSettings needed) | unchecked (`false`) |
| Sync width & height | `L["Sync width & height"]` | `DB.syncSize` | Write `DB.syncSize = checked`; call `SetIconSize(DB.width, DB.height)` to re-mirror; refresh height-slider dim state | checked (`true`) |

Label `GameFontHighlight` FontString anchored `LEFT, check, RIGHT, 4, 0`.
Source: CONTEXT.md D-08, D-09; PLH `CreateCheckbox` (line 629).

## Interaction Contract

### Open / Close

| Trigger | Behavior |
|---------|----------|
| `/plbeast` (bare command, not in combat) | If frame does not exist: lazy-create it, then show. If frame exists and visible: hide it. If frame exists and hidden: show it. |
| `/plbeast` during `InCombatLockdown()` | Block open. Set `pendingOptionsOpenAfterCombat = true`. Print `L["Cannot open options in combat."]` to chat. |
| `PLAYER_REGEN_ENABLED` while `pendingOptionsOpenAfterCombat = true` | Clear flag. Open frame. |
| Click `BasicFrameTemplateWithInset` X button | Frame hides (built-in behavior). |

**Source:** CONTEXT.md D-06, D-07; PLH `ToggleOptions` (line 1021); PLH lines 321, 2623‚Äì2626.

### Live Apply

All control changes take effect immediately on the icon with no close/reload required.

| Control | Immediate effect |
|---------|-----------------|
| Width slider | `SetIconSize(value, DB.height)` ‚Äî icon resizes in real time |
| Height slider | `SetIconSize(DB.width, value)` ‚Äî icon resizes in real time |
| Border thickness slider | Write `DB.borderThickness = value`, `ApplyIconSettings()` |
| Border color swatch | `swatchFunc`/`opacityFunc` write `DB.borderColor`, `ApplyIconSettings()` |
| Lock position checkbox | Write `DB.locked` ‚Äî drag scripts already read it; no setter call needed |
| Sync width & height checkbox | Write `DB.syncSize`, call `SetIconSize(DB.width, DB.height)`, refresh height-slider dim |

**Source:** CONTEXT.md D-11, D-12.

### Combat Nuance

The options frame may remain open if combat starts while it is already open. Only
**opening** the frame is combat-gated. Control interaction while the frame is open during
combat is allowed because the PLBeast root frame is non-secured (`CreateFrame("Frame")`)
and `SetSize`/`SetBackdrop` are not restricted by combat lockdown.
**Source:** CONTEXT.md D-12.

### Slider Initialization

When the frame is created (first open), seed each slider from the current `DB` value:
- Width slider: `slider:SetValue(DB.width or 40)`
- Height slider: `slider:SetValue(DB.height or 40)`
- Border thickness slider: `slider:SetValue(DB.borderThickness or 1)`

When the frame is subsequently shown (toggle show after hide), sliders already reflect
the current `DB` state because every live-apply write updates `DB` before the setter is
called. No re-seed is needed on show.

### Checkbox Initialization

Seed each checkbox on frame creation:
- Lock position: `check:SetChecked(DB.locked or false)`
- Sync width & height: `check:SetChecked(DB.syncSize ~= false)` (default true)

## Slash Command Routing

Replace the current bare-command `else` branch in `SlashCmdList["PLBEAST"]` with frame
open/toggle. Keep `debug` and `reset` subcommands unchanged.

```
msg == "debug"   ‚Üí toggle DB.debug, print confirmation   (unchanged)
msg == "reset"   ‚Üí reset rotation to wyvern              (unchanged)
msg == ""        ‚Üí open/toggle options frame (new, combat-deferred)
anything else    ‚Üí print usage line: L["PLBeast. /plbeast | debug | reset"]
```

**Source:** CONTEXT.md D-06.

## Frame Lifecycle Contract

1. `eventFrame` receives `PLAYER_LOGIN` ‚Üí `CreateBeastIcon()` runs ‚Üí `root` assigned.
2. `PLAYER_REGEN_ENABLED` is registered in the `PLAYER_LOGIN` block alongside existing
   events (needed for deferred-open). No new event frame required; add to the existing
   `eventFrame` registrations.
3. First `/plbeast` call ‚Üí `ToggleOptions()` creates `optionsFrame` lazily, seeds
   controls from `DB`, shows frame.
4. Subsequent `/plbeast` calls ‚Üí `optionsFrame:IsShown()` branch ‚Üí show or hide.
5. Frame can be dragged anywhere on screen (`BasicFrameTemplateWithInset` provides
   drag wiring automatically via its template scripts).
