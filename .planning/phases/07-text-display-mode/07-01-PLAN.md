<objective>
Add an alternative text-based display mode to PLBeast that shows the predicted next beast as a colored text label instead of an icon texture, with per-beast customizable colors.

Purpose: Give players who prefer text over icons a clean, colored text label showing the next beast name, togglable from the options frame. Beast colors default to a colorblind-safe palette but are fully customizable via color pickers.
Output: Modified PLBeast.lua with dual-widget display (tex + label on root frame), mode toggle, font size control, per-beast color pickers, and updated enUS.lua locale strings.
</objective>

<task type="auto">
  <name>Task 1: Core text display — FontString, color table, ApplyDisplayMode, SetNextBeastId extension</name>
  <files>PLBeast/PLBeast.lua</files>
  <action>
  This task adds the rendering infrastructure for text mode. All changes are in PLBeast.lua.

  1. Add DEFAULT_BEAST_COLORS table immediately after the existing BEAST_LABEL_BY_ID table (after line 80). Use Okabe-Ito colorblind-safe RGB values normalized to 0-1 (TEXT-02 defaults):
     - wyvern: { r = 0.337, g = 0.706, b = 0.914 } (sky blue, hex 56B4E9)
     - boar: { r = 0.902, g = 0.624, b = 0.000 } (orange, hex E69F00)
     - bear: { r = 0.000, g = 0.620, b = 0.451 } (bluish green, hex 009E73)
     Use r/g/b keys (not array indices) to match the existing DB.borderColor convention.

  2. Add a helper function GetBeastColor(beastId) that returns r, g, b for a given beast. It reads from DB.textColors[beastId] if present, falling back to DEFAULT_BEAST_COLORS[beastId]. This centralizes color resolution so both SetNextBeastId and the options frame use the same logic.

  3. Add three new keys to the defaults table (around line 14): textMode = false, fontSize = 16, textColors = nil. textColors is nil by default (meaning use DEFAULT_BEAST_COLORS); when the user customizes a color it becomes a table like { wyvern = {r,g,b}, boar = {r,g,b}, bear = {r,g,b} }.

  4. Add a module-level forward declaration for the custom font object: local textFont. Place it near the other forward declarations (around line 133, near the root forward declaration).

  5. In CreateBeastIcon(), after the existing root.tex setup and before the border edge creation:
     - Create the custom Font object: textFont = CreateFont("PLBeastTextFont"). Seed it from GameFontNormalLarge to inherit the locale-correct font file: call GetFont on the GameFontNormalLarge global font object to get the file path and flags, then call textFont:SetFont with that file, DB.fontSize or 16, and those flags.
     - Create root.label as a FontString: f:CreateFontString(nil, "OVERLAY", "GameFontNormalLarge"). Anchor it centered on f via SetPoint("CENTER", f, "CENTER", 0, 0). Call SetJustifyH("CENTER"). Assign the custom font object via label:SetFontObject(textFont). Set initial text from BEAST_LABEL_BY_ID[nextBeastId] and initial color from GetBeastColor(nextBeastId) using SetTextColor(r, g, b). Store as f.label = label.

  6. Create a local function ApplyDisplayMode() placed between ApplyIconSettings and CreateBeastIcon. This function:
     - Guards: if not root then return end
     - Reads local textMode = DB.textMode
     - Calls root.tex:SetShown(not textMode)
     - Calls root.label:SetShown(textMode) (guard: if root.label)
     - When textMode is true: iterate root.borderEdges and call :Hide() on each edge texture (border makes no visual sense around free text). When textMode is false: call ApplyIconSettings() to restore border visibility correctly based on DB.borderThickness (do NOT call SetShown(true) on edges directly — that would regress users who set borderThickness to 0).
     - When textMode is true and root.label exists: also refresh the label color from GetBeastColor(nextBeastId) so that any recent color picker change is reflected.

  7. Extend SetNextBeastId (around line 178). After the existing root.tex:SetTexture block, add a parallel block: if root and root.label then set root.label text from BEAST_LABEL_BY_ID[nextBeastId] (fallback to BEAST_LABEL_BY_ID.boar), and get r, g, b from GetBeastColor(nextBeastId) and call SetTextColor(r, g, b).

  8. At the end of CreateBeastIcon(), after the call to ApplyIconSettings(), call ApplyDisplayMode() so the correct widget is visible on initial frame creation.

  9. In the ADDON_LOADED handler, add coercion for the new DB fields alongside the existing coercions (around line 738):
     - DB.fontSize = tonumber(DB.fontSize) or 16
     - if DB.textMode == nil then DB.textMode = false end
     - Validate DB.textColors: if it exists and is a table, ensure each beast entry (if present) has r/g/b number keys; if malformed, nil it out. If DB.textColors is not nil and not a table, set it to nil.
  </action>
