<objective>
Add an alternative text-based display mode to PLBeast that shows the predicted next beast as a colored text label instead of an icon texture.

Purpose: Give players who prefer text over icons a clean, colorblind-safe text label showing the next beast name, togglable from the options frame or slash command.
Output: Modified PLBeast.lua with dual-widget display (tex + label on root frame), mode toggle, font size control, and updated enUS.lua locale strings.
</objective>

<task type="auto">
  <name>Task 1: Core text display — FontString, color table, ApplyDisplayMode, SetNextBeastId extension</name>
  <files>PLBeast/PLBeast.lua</files>
  <action>
  This task adds the rendering infrastructure for text mode. All changes are in PLBeast.lua.

  1. Add BEAST_COLOR_BY_ID table immediately after the existing BEAST_LABEL_BY_ID table (after line 80). Use Okabe-Ito colorblind-safe RGB values normalized to 0-1 (TEXT-02):
     - wyvern: { 0.337, 0.706, 0.914 } (sky blue, hex 56B4E9)
     - boar: { 0.902, 0.624, 0.000 } (orange, hex E69F00)
     - bear: { 0.000, 0.620, 0.451 } (bluish green, hex 009E73)

  2. Add two new keys to the defaults table (around line 14): textMode = false, fontSize = 16. These follow the same flat-defaults merge convention as every existing key.

  3. Add a module-level forward declaration for the custom font object: local textFont. Place it near the other forward declarations (around line 133, near the root forward declaration).

  4. In CreateBeastIcon(), after the existing root.tex setup and before the border edge creation:
     - Create the custom Font object: textFont = CreateFont("PLBeastTextFont"). Seed it from GameFontNormalLarge to inherit the locale-correct font file: call GetFont on the GameFontNormalLarge global font object to get the file path and flags, then call textFont:SetFont with that file, DB.fontSize or 16, and those flags.
     - Create root.label as a FontString: f:CreateFontString(nil, "OVERLAY", "GameFontNormalLarge"). Anchor it centered on f via SetPoint("CENTER", f, "CENTER", 0, 0). Call SetJustifyH("CENTER"). Assign the custom font object via label:SetFontObject(textFont). Set initial text from BEAST_LABEL_BY_ID[nextBeastId] and initial color from BEAST_COLOR_BY_ID[nextBeastId] using SetTextColor(c[1], c[2], c[3]). Store as f.label = label.

  5. Create a local function ApplyDisplayMode() placed between ApplyIconSettings and CreateBeastIcon. This function:
     - Guards: if not root then return end
     - Reads local textMode = DB.textMode
     - Calls root.tex:SetShown(not textMode)
     - Calls root.label:SetShown(textMode) (guard: if root.label)
     - Hides border edges when text mode is active: iterate root.borderEdges and call SetShown(not textMode) on each edge texture (per RESEARCH Pitfall 4 recommendation — border makes no visual sense around free text). When not in text mode, do NOT force-show edges here — let ApplyIconSettings handle border visibility via its thickness logic.

  6. Extend SetNextBeastId (around line 178). After the existing root.tex:SetTexture block, add a parallel block: if root and root.label then set root.label text from BEAST_LABEL_BY_ID[nextBeastId] (fallback to BEAST_LABEL_BY_ID.boar), and set color from BEAST_COLOR_BY_ID[nextBeastId] (fallback to BEAST_COLOR_BY_ID.boar) using SetTextColor with the three RGB values from the table entry.

  7. At the end of CreateBeastIcon(), after the call to ApplyIconSettings(), call ApplyDisplayMode() so the correct widget is visible on initial frame creation.

  8. In the ADDON_LOADED handler, add coercion for the new DB fields alongside the existing coercions (around line 738): DB.fontSize = tonumber(DB.fontSize) or 16. The textMode field is boolean so no numeric coercion is needed, but ensure it defaults: if DB.textMode == nil then DB.textMode = false end (although the flat-defaults merge handles this, an explicit guard is consistent with the borderColor guard pattern).
  </action>
