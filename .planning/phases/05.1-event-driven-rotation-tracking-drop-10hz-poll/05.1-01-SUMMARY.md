# Phase 05.1 Plan 01: Detection Engine Rewrite Summary

**One-liner:** Replaced 10Hz C_Timer.NewTicker poll with AzortharionUI's self-correcting CDM model — per-frame OnUpdate detection, lazy frame re-resolve, debounced rebuild, and string-based SavedVariables persistence.

## What Was Built

Complete rewrite of PLBeast.lua's detection engine to architecturally match AzortharionUI Hunter.lua (v7.5), implementing all decisions D-01 through D-10.

### New Functions Added

| Function | Purpose |
|----------|---------|
| `PollPackLeader()` | Self-correcting prediction state machine (Azor lines 245-326 adapted) |
| `CDMFrameHasAura(data)` | Lazy per-entry CDM frame re-resolve + auraInstanceID check |
| `ScheduleRebuild()` | 0.5s debounced CDM cache rebuild via C_Timer.NewTimer |
| `SaveState()` | Write `DB.plNextBeastId` to SavedVariables |
| `RestoreState()` | Read `DB.plNextBeastId` with legacy `DB.nextIndex` migration |
| `ClearPackLeaderState()` | Reset state machine vars (plPhase, plBeast, plHasCdBuff) |
| `GetFrameCooldownID(frame)` | Extract cooldownID from frame or frame.cooldownInfo |
| `MatchTrackedSpell(info)` | Check spellID + overrideSpellID + linkedSpellIDs vs TRACKED_SPELL_IDS |
| `TryCache(frame)` | Cache entry point using MatchTrackedSpell |
| `ScanDeep(frame, depth)` | Recursive DFS frame scanner (depth cap 3) |
| `SearchFrameForCooldownID(frame, targetID, depth)` | DFS search for a specific cooldownID (depth cap 4) |
| `OnUpdateHandler()` | OnUpdate callback with GetTime() equality throttle |

### Variables Removed

`prevReady`, `pollTicker`, `POLL_INTERVAL`, `CDM_REBUILD_COOLDOWN`, `lastCDMRebuildAt`, `CDM_CATEGORY_INDEXES`, `ID_BY_INDEX`, `INDEX_BY_ID`

### Key Behavioral Changes

1. **Detection mechanism**: `UNIT_AURA` event registration removed; OnUpdate drives `PollPackLeader()` at ~1Hz (GetTime equality guard)
2. **Prediction model**: Pin to ready beast on buff appear; advance to `NEXT_BEAST[id]` on buff consume (self-correcting)
3. **Icon semantic**: Shows the ready beast itself during ready phase (boar ready = boar icon), not the beast after
4. **Default beast**: boar (was wyvern)
5. **Persistence**: `DB.plNextBeastId` (string) with `DB.nextIndex` legacy migration in ADDON_LOADED
6. **No login/boss-pull reset**: `PLAYER_ENTERING_WORLD` and `ENCOUNTER_START` not registered
7. **CDM rebuild events**: Three triggers (`COOLDOWN_VIEWER_DATA_LOADED`, `COOLDOWN_VIEWER_SPELL_OVERRIDE_UPDATED`, `CooldownViewerSettings.OnDataChanged`) all route through `ScheduleRebuild()`
8. **Combat-enter fallback**: `PLAYER_REGEN_DISABLED` calls `BuildCDMCache()` if cache empty
