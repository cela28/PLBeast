<objective>
Rewrite PLBeast's rotation tracking engine to match AzortharionUI's event-driven, self-correcting detection model (D-01 through D-10, PERF-01).

Purpose: Eliminate the 10Hz C_Timer.NewTicker poll; adopt per-frame OnUpdate with GetTime throttle (zero cost when idle). Switch from snapshot-diff to Azor's self-correcting prediction model (pin to ready beast, advance on consumption). This makes PLBeast's predictions match AzortharionUI step-for-step.

Output: A rewritten PLBeast.lua whose detection model is architecturally equivalent to AzortharionUI Hunter.lua lines 33-326 + 411-475, with all Phase 2 dead code (timer poll, snapshot-diff, C_UnitAuras fallback) removed.
</objective>

## Phase Goal

**As a** Pack Leader hunter, **I want to** see PLBeast's next-beast prediction match AzortharionUI's step-for-step with no constant polling, **so that** I get accurate predictions with negligible idle CPU cost.

## Artifacts this phase produces

**New functions (PLBeast.lua):**
- `MatchTrackedSpell(info)` -- spellID/overrideSpellID/linkedSpellIDs matching for CDM cache
- `TryCache(frame)` -- cache entry point using MatchTrackedSpell
- `ScanDeep(frame, depth)` -- recursive DFS frame scanner (replaces BFS)
- `CDMFrameHasAura(data)` -- lazy per-entry CDM frame re-resolve + auraInstanceID check
- `ScheduleRebuild()` -- 0.5s debounced CDM cache rebuild timer
- `PollPackLeader()` -- self-correcting prediction model (replaces CheckAuraState)
- `ClearPackLeaderState()` -- reset state machine vars
- `SaveState()` -- write DB.plNextBeastId to SavedVariables
- `RestoreState()` -- read DB.plNextBeastId from SavedVariables with legacy migration
- `GetFrameCooldownID(frame)` -- extract cooldownID from frame or frame.cooldownInfo
- `SearchFrameForCooldownID(frame, targetID, depth)` -- recursive DFS search for a specific cooldownID

**New state variables (PLBeast.lua):**
- `plPhase` (string: 'off'|'ticking'|'ready')
- `plBeast` (table or nil: current ready beast data)
- `plHasCdBuff` (boolean: countdown buff active)
- `plDirty` (boolean: change flag for future UI consumers)
- `lastPolledTime` (number: GetTime throttle for OnUpdate)
- `rebuildTimer` (timer handle for ScheduleRebuild)

**New constants (PLBeast.lua):**
- `PL_COUNTDOWN = 471877`
- `PL_BEASTS` (ordered array: boar, bear, wyvern -- matching Azor's scan order)
- `CDM_CATS` (Enum.CooldownViewerCategory set)

**Modified functions:**
- `BuildCDMCache()` -- rewritten to use ScanDeep + TryCache + CDM_CATS category scan
- `FindCDMFrameByCooldownID()` -- searches ALL CDM_FRAME_NAMES (not just CDMGroups_Buffs)
- `RefreshVisibility()` -- wires OnUpdate instead of poll ticker; removes UNIT_AURA registration
- `SetNextBeastId()` -- writes DB.plNextBeastId; default fallback changes to "boar"
- `ADDON_LOADED` handler -- string-based persistence migration from DB.nextIndex to DB.plNextBeastId

**Removed functions:**
- `CheckAuraState()`, `SyncNextFromAddedReady()`, `SeedAuraSnapshot()`, `IsReadyViaAura()`, `IsReadyViaCDM()`, `IsReadyBuffActive()`, `StartPollTicker()`, `StopPollTicker()`, `ResetRotationToWyvern()`, `EnsureCacheFramesResolved()`, `FrameStillOwnsCooldown()`, `GatherCooldownsFromFrameTree()`, `GatherCooldownsFromViewerCategories()`, `RegisterDiscoveredCooldown()`, `ReadSpellFromCooldownID()`, `NormalizeNextIndex()`

**Removed variables:**
- `pollTicker`, `POLL_INTERVAL`, `CDM_REBUILD_COOLDOWN`, `lastCDMRebuildAt`, `prevReady`

**Modified locale keys:**
- `"Rotation reset. Next: Wyvern."` changed to `"Rotation reset. Next: Boar."`
