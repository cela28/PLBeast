## Phase Goal (User Story)

**As a** Pack Leader hunter, **I want to** drag the beast icon where I want it and have its position, size, and border persist across sessions, **so that** my HUD layout stays exactly how I set it every time I log in.

<objective>
Complete the Phase 4 icon by adding interaction and full persistence on top of Plan 01's visible bordered icon: left-button drag-to-reposition with a silent combat-lockdown guard and center-offset persistence (UI-03, UI-04), and an `ApplyIconSettings()` function that applies persisted size (independent width/height via SetSize), position, and border (thickness + color) from `PLBeastDB` on login (UI-05, UI-06, UI-07). A sync-aware sizing helper honors `DB.syncSize` so Phase 5's sliders can mirror width↔height.

Purpose: After this plan, a real user can move the icon and trust that position, size, and border all survive `/reload` and logout — the remaining user-facing capability of the phase.

Output: `OnDragStart`/`OnDragStop` scripts on PLBeastFrame, a `SetIconSize(w, h)` sync-aware helper, an `ApplyIconSettings()` function, and refactoring of CreateBeastIcon/PLAYER_LOGIN to apply settings through ApplyIconSettings.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SavedVariables → addon | `PLBeastDB` position/size/border values are player-editable on disk; applied untrusted on login via ApplyIconSettings |
| User input (drag) → frame | Mouse drag mutates frame position; must be gated by combat lockdown |
| Combat lockdown → frame mutation | WoW taints/blocks protected frame moves during combat; drag must be guarded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Elevation of Privilege | frame move during combat lockdown | mitigate | `OnDragStart` returns silently when `InCombatLockdown()` is true (D-08); StartMoving never called in combat (Task 1) |
| T-04-05 | Tampering | stale LayoutCache vs SavedVariables double-source | mitigate | `SetUserPlaced(false)` after StopMovingOrSizing prevents WoW writing a competing position to LayoutCache.txt (Pitfall 2, Task 1) |
| T-04-06 | Denial of Service | `GetLeft()` nil before frame rendered | mitigate | `if not self:GetLeft() then return end` nil guard in OnDragStop prevents nil-arithmetic crash (Pitfall 3, Task 1) |
| T-04-07 | Tampering | malformed numeric `DB.width/height/borderThickness` from edited SavedVariables | accept | `or 40` / `or 1` fallbacks handle nil; a non-numeric edit would error in SetSize but is a low-risk player self-edit with no external attack surface. Phase 5 sliders clamp to documented ranges (16-128, 0-8) |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | Not applicable — WoW addon, no package manager, no installs (RESEARCH.md confirms) |
</threat_model>
