## Phase Goal (User Story)

**As a** Pack Leader hunter, **I want to** see a single on-screen icon showing the next beast in my rotation with a clean bordered look, **so that** I always know what is coming next at a glance.

<objective>
Deliver the smallest end-to-end visible slice of the Phase 4 icon: a single named frame, created eagerly at PLAYER_LOGIN, that displays the correct beast texture (wyvern/boar/bear) driven by the existing `nextBeastId` state, updates live as the rotation advances, shows/hides via the Phase 3 `isPackLeaderActive` flag, and renders a configurable square border (default black, 1px) using BackdropTemplate.

Purpose: After this plan, a real Pack Leader hunter sees a working bordered icon on screen that reflects the rotation prediction — the core visible value of the phase. Drag and resize are added in Plan 02.

Output: Extended `defaults` table, `ICON_FILE_BY_ID` constants, `local root` forward declaration, `CreateBeastIcon()`, a texture-push hook in `SetNextBeastId()`, a `root:SetShown(isPackLeaderActive)` bridge in `RefreshVisibility()`, a `CreateBeastIcon()` call in PLAYER_LOGIN, and a borderColor deep-copy guard in ADDON_LOADED.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SavedVariables → addon | `PLBeastDB` values (offsetX, width, borderColor, etc.) are player-editable on disk and loaded untrusted at ADDON_LOADED |
| WoW client → addon frame | Combat lockdown state and frame-render timing can change between calls |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | `DB.borderColor` table-valued default | mitigate | Deep-copy guard in ADDON_LOADED ensures DB.borderColor is a distinct table; `SetBackdropBorderColor` reads with `0,0,0,1` fallbacks so a nil/partial table cannot crash border setup (Task 1, Task 2) |
| T-04-02 | Denial of Service | `root.tex:SetTexture` in SetNextBeastId before frame exists | mitigate | `if root and root.tex then` guard prevents nil-index crash when SetNextBeastId runs at ADDON_LOADED (Pitfall 5, Task 2) |
| T-04-03 | Tampering | malformed `DB.width`/`DB.height` from edited SavedVariables | accept | `SetSize(DB.width or 40, DB.height or 40)` falls back on nil; a non-numeric value would error but is low-risk (player self-edit only, no external input). Phase 5 sliders will clamp ranges (16-128) |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | Not applicable — WoW addon, no package manager, no installs. No legitimacy audit needed (RESEARCH.md confirms) |
</threat_model>
