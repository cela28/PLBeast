| Property | Value | Source |
|----------|-------|--------|
| Drag enabled by default | Yes â€” `DB.locked = false` | 04-CONTEXT.md D-07 |

| Clamp to screen | `frame:SetClampedToScreen(true)` | PLH pattern |
| OnDragStart | `if InCombatLockdown() then return end` â€” silent no-op; then `self:StartMoving()` | 04-CONTEXT.md D-08 |
| OnDragStop | `self:StopMovingOrSizing()` â†’ capture new center offset â†’ persist to `DB.offsetX/offsetY` | 04-CONTEXT.md D-09 |
| Combat guard | `InCombatLockdown()` check at top of `OnDragStart`; no chat message, no error | 04-CONTEXT.md D-08 |

**Offset capture contract (OnDragStop):**

```lua
frame:SetScript("OnDragStop", function(self)
    self:StopMovingOrSizing()
    local cx = UIParent:GetWidth()  / 2
    local cy = UIParent:GetHeight() / 2
    local fx = self:GetLeft() + self:GetWidth()  / 2
    local fy = self:GetBottom() + self:GetHeight() / 2
    DB.offsetX = fx - cx
    DB.offsetY = fy - cy
end)
```

### Resize Behavior (UI-05, UI-07)

- Resize is not interactively triggered in Phase 4 (no sliders exist until Phase 5).
- The frame must be sized correctly at creation and after a reload using the persisted `DB.width` / `DB.height` values.
- Sync toggle is consulted only when a slider changes a value (Phase 5); in Phase 4 it affects only the default initialization: both default to 40.
