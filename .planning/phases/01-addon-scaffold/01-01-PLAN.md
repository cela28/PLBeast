<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SavedVariables file -> addon | PLBeastDB is read from disk by WoW client; addon trusts the data |
| Slash command input -> handler | User-typed /plbeast arguments are untrusted strings |
| Locale global -> main file | PLBeastLocale global could be polluted by another addon |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | PLBeastDB (SavedVariables) | accept | WoW sandbox restricts file access; only the WoW client writes SV files. Flat merge loop only fills nil keys, so unexpected keys are harmless but not sanitized. Low risk -- no PII, addon-local config only |
| T-01-02 | Tampering | PLBeastLocale global | mitigate | setmetatable(PLBeastLocale or {}, ...) -- the or {} guard prevents nil dereference if another addon deletes the global before PLBeast.lua loads; __index fallback returns the key string for any missing entry |
| T-01-03 | Information Disclosure | dprint output | accept | Debug output is gated by DB.debug (default false); output goes only to the player's own chat frame. No sensitive data exposed -- just debug=false and nextIndex=1 |
| T-01-04 | Denial of Service | Slash command handler | mitigate | msg is trimmed and lowercased before use; Phase 1 handler does not execute arbitrary input -- it always prints the same fixed message regardless of input |
| T-01-05 | Spoofing | Global namespace | mitigate | Only three globals created: PLBeastDB (required by SavedVariables), PLBeastLocale (set by locale file), SLASH_PLBEAST1 (required by WoW slash system). All other state is file-local. No accidental global leaks from missing local keyword |
</threat_model>
